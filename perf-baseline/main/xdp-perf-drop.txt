# XDP perf baseline — DROP mode
#
# Slice 07 / step 07-02 — placeholder baseline seeded for the
# `cargo xtask xdp-perf` regression gate. The first follow-on PR
# that runs `xdp-bench drop` against the loaded program inside Lima
# (or CI) overwrites this file with real measurements.
#
# Why a placeholder?
#
# The gate's pure decision fn is delta-only per
# `.claude/rules/testing.md` § "Tier 4 — Verifier and Performance
# Gates":
#
#   "Never gate on absolute numbers — runner hardware varies enough
#    to make absolute gates flaky. Deltas only."
#
# Until a real measurement lands, the recorded values represent a
# realistic ballpark for the runner class named under
# `# Recorded against:` so that any future measurement compares
# against a baseline that is approximately right. The gate fires on
# >5% pps drop AND/OR >10% p99 latency rise — recording an order-of-
# magnitude wrong placeholder would either silence the gate (false
# positives go undetected) or fire it spuriously on the first real
# run. The values below are the realistic ballpark.
#
# DROP mode is XDP_DROP fast path — no map lookup, no rewrite. Tests
# the verifier+JIT cost of the program prologue alone. On commodity
# Lima/virtio-net at single-queue, this should sit in the
# 10–15 Mpps range; on bare metal mlx5/i40e, 20–40 Mpps is typical.
#
# Recorded against:
#   kernel:  6.8.0-111-generic   (Lima default; matrix row "6.6 LTS")
#   commit:  step 07-02 — initial placeholder
#   date:    2026-05-07
#   tool:    xdp-bench drop <iface>
#   target:  target/xtask/bpf-objects/overdrive_bpf.o (xdp_pass program;
#            real LB program produces a different baseline once the
#            xdp-bench harness is wired in CI)
#   runner:  Lima virtio-net single queue
#
# Gate thresholds (see xtask/src/perf_gate/xdp_perf.rs::GatePolicy):
#   pps    — fail if measured < baseline * (1 - 0.05) (i.e. >5% drop)
#   p99_ns — fail if measured > baseline * (1 + 0.10) (i.e. >10% rise)
#
# Format (one line per mode, space-separated key=value pairs;
# step 07-02's parser keys off `mode=`, `pps=`, `p99_ns=`):

mode=drop pps=12500000 p99_ns=1800
