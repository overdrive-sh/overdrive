# Verifier-budget baseline — xdp_reverse_nat_lookup
#
# Step 06-05 (this commit) — FIRST baseline for the new
# `xdp_reverse_nat_lookup` XDP program landed in slice 09-02 per
# ADR-0045 § 6 (Verifier-budget envelope) / § 8 (Phasing).
#
# Slice 07's `cargo xtask verifier-regress` gate (lands at step
# 07-01) parses this file and fails the PR if the recorded program
# exceeds the baseline by >20% or approaches the per-program
# complexity ceiling (1,000,000 instructions for CAP_BPF programs)
# by >10%, per ASR-2.2-03.
#
# Program shape (post-pivot):
#   1. sanity prologue (shared with xdp_service_map_lookup)
#   2. REVERSE_NAT_MAP lookup keyed on (client_ip, client_port,
#      backend_ip, backend_port, proto) — BPF_MAP_TYPE_HASH
#   3. L3+L4 rewrite (backend_ip, backend_port → VIP, vip_port)
#      with incremental csum
#   4. bpf_fib_lookup against post-rewrite (src_ip, dst_ip)
#   5. L2 MAC rewrite + bpf_redirect_neigh(ifindex, NULL, 0, 0)
#      on FIB hit; XDP_PASS on REVERSE_NAT miss or FIB miss
#
# The reverse-NAT path lands on the post-pivot bpf_redirect_neigh
# shape from the start (slice 09-02). It does NOT carry the
# word-by-word csum recomputation that xdp_service_map_lookup needs:
# the reverse path's incremental csum is sufficient because the
# rewrite is symmetric to the original incremental NAT — see
# ADR-0040 Q1=A and ADR-0045 § Decision § 3 (single-cut greenfield
# migration; the retired TC-egress reverse-NAT did NOT land on the
# pre-pivot shape, so no migration of historical baselines exists).
#
