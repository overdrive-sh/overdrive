#     Delta vs Slice 03: 460 → 660 (+200 instructions, +43.5%).
#

#   Slice 05-04 (this commit): verified_insns = 1211 with
#     `bpf_fib_lookup` + L2 MAC rewrite + cross-iface `bpf_redirect`
#     appended after the existing L3+L4 rewrite + checksum fold
#     (Option α per
#     docs/research/dataplane/cilium-bpf-fib-lookup-l2-mac-rewrite-comprehensive-research.md).
#     The added instructions: ~80 bytes of `bpf_fib_lookup` parameter
#     init from packet fields (tot_len, tos, sport, dport, src_ip,
#     dst_ip, ifindex, family, proto), one helper call (`bpf_fib_lookup`,
#     helper id 69), the success-path conditional branching on

#     bounds checks, an `ingress_ifindex == fib.ifindex` comparison,
#     and the `bpf_redirect` helper call in the cross-iface path.
#
#     Delta vs Slice 04 baseline: 660 → 1211 (+551 instructions,
#     +83%). This exceeds the 20% relative-delta gate against Slice
#     04's baseline, so the gate is being re-baselined here per
#     ADR-0040 / architecture.md § 7 D3 — Option α is the load-bearing
#     L2 MAC rewrite + redirect mechanic that unblocks S-2.2-17 (real
#     `nc` end-to-end through 3-iface topology). Without it, every
#     `XDP_TX` into a non-local backend's veth peer is dropped at
#     `ip_rcv` due to `PACKET_OTHERHOST` classification — see Findings
#     2.4 / 3.3 of the cited research.
#
#     1211 instructions remains comfortably within every absolute
#     ceiling: 0.24% of the 500K L1-cache-fits target, 0.20% of the
#     600K ASR-2.2-03 ceiling, 0.12% of the 1M CAP_BPF kernel ceiling.
#     The +20% gate now enforces against THIS new baseline (1211),
#     i.e. `verified_insns ≤ 1453` for future changes.
#
#     The pps cost of the added helper call is predicted by the
#     research at ≤ 2% (Finding 4.2); empirical measurement lands in
