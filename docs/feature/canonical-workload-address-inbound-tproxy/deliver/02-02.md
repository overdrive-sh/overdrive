## D3 (medium) — all-mesh retry-bump guard: ratified + pinned

**Decision (orchestrator-ratified):** the guard
`if !(local_is_empty && remote_is_empty) { …record fingerprint / bump retries… }` — which makes
a fully-gated all-mesh service record NOTHING in the `View` — is **kept** as the intended
contract.

**Trace (the reviewer's own adversarial assessment, which the decision adopts):** a fully-gated
mesh service has no convergence target in this reconciler (nft-TPROXY owns Path-A delivery).
*Without* the guard it would record `last_attempted_fingerprint = Some(fp)` — an affirmative lie
that it "programmed at fp" — and, because `backoff_for_attempt` is degenerate-constant at 1 s,
re-dispatch and **fsync a `ViewStore` row ~once/second forever**. *With* the guard: zero actions,
zero `ViewStore` I/O, no phantom fingerprint — only a cheap pure recompute each tick. The guard
is strictly **more honest**, not "worse." It is a behavioral decision over a corner the design
left open (what a fully-gated service records), **not** new public API surface (no new method /
type / variant / param — so not the `TerminalErrorKind::Retryable` precedent class); the
milder form is ratified here and converted into a pinned contract rather than left implicit.

**Pinned by:** `all_mesh_service_emits_nothing_and_keeps_retries_empty_across_ticks`
(`mesh_backend_lb_gate.rs`, `92702af5`) — two consecutive `reconcile` ticks over an all-mesh
service (`[10.99.0.6, 10.99.0.10]`, view fed forward, later `now_unix`), asserting
`actions.is_empty()` AND `next_view.retries.is_empty()` on **both** ticks. The mutation run
confirms this test is load-bearing: both `:426` guard mutants are caught by it.
